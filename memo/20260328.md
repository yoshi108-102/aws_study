とりあえずここだけ使って、めちゃくちゃ単純なAPI GateWayを試すためのコードを作る。
とりあえず動かすことには成功した。
GET hello および POST echoに対応したAPIを作成し、GateWay経由でlambdaを呼び出している。

AWS Powertoolsを使うとログ管理がしやすいらしいので試してみる

今回やったことは、要するに「Lambda の print ログを、Powertools の構造化ログに置き換えて CloudWatch で追いやすくした」です。

具体的には次の4点です。

LambdaにPowertools Layerを追加
main.tf で aws_lambda_function.api_handler に layers を追加しました。
これでデプロイ後の Lambda 実行環境で aws_lambda_powertools が import できます。
Powertools用の環境変数を追加
main.tf の environment.variables に
POWERTOOLS_SERVICE_NAME
POWERTOOLS_LOG_LEVEL
を追加しました。
ログレベルを Terraform 変数で切り替えられるようにしています。
handlerをPowertools Logger化
api.py で
Logger() を導入
@logger.inject_lambda_context(...) を追加
handler_start, route_matched_*, echo_body_parse_*, handler_end などを logger.info/warning で出すように変更
さらに append_keys(method, path) で、各ログに HTTP 情報を共通付与しています。
権限エラー回避のためLayer参照方式を変更
最初は Terraform の data aws_lambda_layer_version で Layer を自動取得しようとしましたが、あなたのIAM権限だと lambda:ListLayerVersions で403になりました。
そのため main.tf で locals に Layer ARNを組み立てる方式へ変更しました（region + version）。
これで追加権限なしで plan/apply できます。
あわせて、variables.tf と README.md も更新して、ログレベル変更や Logs Insights クエリをすぐ試せるようにしています。

```python
@logger.inject_lambda_context(
    log_event=False,
    correlation_id_path=correlation_paths.API_GATEWAY_HTTP,
    clear_state=True,
)
def lambda_handler(event, context):
    method = event.get("requestContext", {}).get("http", {}).get("method", "")
    path = event.get("rawPath", "")

    logger.append_keys(method=method, path=path)
    logger.info("handler_start")

    if method == "GET" and path.endswith("/hello"):
        logger.info("route_matched_hello")
        response = _response(200, build_hello_payload(method=method, path=path))
        logger.info("handler_end", extra={"status_code": 200})
        return response

    if method == "POST" and path.endswith("/echo"):
        logger.info("route_matched_echo")
        try:
            body = parse_echo_body(event)
        except ValueError:
            logger.warning("echo_body_parse_failed")
            return _response(400, {"error": "invalid json"})

        logger.info("echo_body_parse_success")
        response = _response(200, {"received": body})
        logger.info("handler_end", extra={"status_code": 200})
        return response

    logger.info("handler_not_found")
    return _response(404, {"error": "not found"})
```
どうやらloggerがものすごく丁寧にjsonになっている！
