Terraform Only / Lambda

AWS Lambda を Terraform だけで作る

このサンプルは、Lambda 関数本体の zip 化から IAM ロール、CloudWatch Logs、Function URL の作成までを Terraform だけで完結させる最小構成です。 まずは 1 本の Lambda を確実に動かし、あとから API Gateway や EventBridge に広げる前段として使えます。

この構成で作るもの

学習しやすい最小セット

Lambda 関数本体は Python で 1 ファイルだけです。Terraform 側で zip を作成し、そのまま Lambda にデプロイします。

公開確認は Function URL を使います。API Gateway を挟まないので、まずは Lambda 自体の実行に集中できます。

CloudWatch Logs を明示的に作ることで、実行ログの確認場所も最初から固定できます。

処理の流れ

apply すると何が起こるか

01
Python ファイルを zip 化

archive provider の archive_file が lambda_function.py を zip にまとめます。

02
Lambda 実行ロールを作成

Lambda が CloudWatch Logs に書き込めるように、基本実行ポリシーをアタッチします。

03
CloudWatch Logs を準備

ロググループを先に作り、保持期間を 7 日に固定します。

04
Function URL を公開

認証なしの URL を作るので、curl やブラウザからすぐ実行確認できます。

Terraform の実行

コマンドはこれだけ

cd tf-lambda-hello
terraform init
terraform plan
terraform apply

curl "$(terraform output -raw function_url)"

tf-lambda-hello/main.tf を読みます。provider、archive_file、IAM、Lambda、Function URL の依存関係を追うと全体像がつかみやすいです。

B
lambda_function.py

tf-lambda-hello/lambda_function.py を読みます。受け取ったイベントから HTTP メソッドとパスを取り出し、JSON を返すだけの処理です。

C
README.md

tf-lambda-hello/README.md には実行手順を短くまとめています。適用後は function_url と cloudwatch_log_group を確認します。

注意点

最初に押さえるべき点

Function URL は認証なしで公開しています。学習用としては分かりやすいですが、長期運用や実サービスでは IAM 認証や API Gateway を検討した方が安全です。

Lambda を理解する最初の段階では、イベントソースを増やしすぎない方がいいです。この構成で 1 回動かした後に、EventBridge や S3 トリガーへ進むのが自然です。
